# Skills catalog

Persona-specific skills referenced by `docs/tasks/*.md` track specs in the loop planning repo. Each entry follows `library + concepts + gotchas` granularity ‚Äî generic categories ("TypeScript expert", "Testing expert") are not acceptable; the persona must point at concrete libraries / domains the implementer must already know.

Entries are categorized so spec authors can scan by area. When adding a new persona, drop it in the closest category ‚Äî create a new category only when 2+ entries would land in it.
